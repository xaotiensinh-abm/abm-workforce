# Goal
Khởi tạo mới, Sao chép phong cách (Imitate), hoặc Chỉnh sửa nội dung Slide PowerPoint (`.pptx`) qua việc sử dụng sức mạnh của Skywork PPT API. Bao gồm các tính năng Local Ops để cắt ghép slide nhanh gọn mà không tốn tải trên server.

# Instructions

## Bước 1: Setup Môi trường
Các công cụ PPT phụ thuộc vào thư viện Python `python-pptx`, cần đảm bảo User đã chạy lệnh cài đặt:
```bash
python3 -m pip install -q --break-system-packages python-pptx
```

## Bước 2: Phân Luồng Yêu Cầu (Routing)
Xác định Ý định User để gọi đúng Script:
1. **Tạo mới (Generate):** Tạo slide mới tinh dựa trên prompt hoặc đọc nội dung web. Nếu có file tham chiếu, chạy `parse_file.py` để lấy ID trước. *(Tham khảo workflow_generate.md)*
2. **Kế thừa mẫu (Imitate):** Chuyển file `.pptx` mẫu của User lên Server lấy URL trước, sau đó tạo slide mới với bố cục y hệt form mẫu. *(Tham khảo workflow_imitate.md)*
3. **Chỉnh sửa bằng Lời (Edit):** Cần url `https://.../file.pptx` đang tồn tại, chạy `run_ppt_write.py` kèm theo câu lệnh (VD: Đổi nền slide 1 sang màu đen). *(Tham khảo workflow_edit.md)*
4. **Local Ops (Offline):** Các tác vụ chỉ cắt, ghép, xóa, đổi chỗ Slide với file local. KHÔNG cần API.
```bash
# Xóa slide số 3, 5
python3 scripts/local_pptx_ops.py delete --file my.pptx --slides 3,5 -o trimmed.pptx
```

## Bước 3: Đợi & Trả Kết Quả
- Đặt `yieldMs: 600000` (10 phút) cho các job tạo mới/sửa từ xa.
- Khi hoàn tất, link CDN và đường dẫn local của File sẽ được xuất ra. Báo về cho User.

# Constraints
- 🚫 **Bảo Mật:** Các tác vụ Tạo/Sửa/Kế thừa sẽ up file tài liệu lên Skywork. Nếu file nội bộ bảo mật cao, không tự ý đẩy lên Server khi chưa báo trước. Nếu thao tác nội dung nhẹ (Cắt/Ghép slide) BẮT BUỘC dùng Layer 4 (Local Ops).
