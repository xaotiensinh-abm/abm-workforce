# Goal

> **Use this skill when:** creating isolated git worktrees for feature development

Tạo các không gian (workspace) song song với kho Git để hỗ trợ cô lập tính năng đang thi công (isolated workspaces) mà không cần switch branch chồng lấn context.

# Instructions

1. **Chọn thư mục (ưu tiên vị trí):**
   - Chạy `ls -d .worktrees` (ưu tiên dạng ẩn) và `ls -d worktrees`.
   - Nếu có sẵn, sử dụng. Nếu không, tham chiếu cấu hình `CLAUDE.md`.
   - Nếu không có nữa, hỏi User liệu nên để `.worktrees/` local hay đẩy lên dạng Global `~/.config/superpowers/worktrees/`. 
2. **Xác minh an toàn Local Gitignore:**
   - BẮT BUỘC: Check `.gitignore` có track nhầm hay không qua lệnh: `git check-ignore -q .worktrees`.
   - Nếu thư mục KHÔNG THẤY ignored, PHẢI THÊM NÓ VÀO `.gitignore` VÀ COMMIT TRƯỚC TIÊN. (Chống track lộn rác của branch khác).
3. **Sinh cấu trúc Worktree:**
   - Tạo nhánh theo template: `git worktree add <path> -b <branch_name>`. Di chuyển cấp vào `.worktrees/...`.
4. **Cài đặt Dependency (Auto Setup):**
   - Nhận diện `package.json` (chạy `npm install`), `Cargo.toml` (`cargo build`), `requirements.txt`/`pyproject.toml` (pip/poetry), `go.mod` (go mod).
5. **Kiểm thử Clean Baseline:**
   - Chạy các test runner mặc định để check dự án ở baseline không mang lỗi di truyền. Fail là phải report ngay.
6. **Report Ready:**
   - Báo cáo Link path.

# Examples

- Lệnh gọi Worktree kiểm tra ignore fail -> Ghi `.worktrees` xuống cuối `.gitignore` -> Commit -> Mới dùng `git worktree add .worktrees/auth -b feature/auth`.
- `cd .worktrees/auth` -> Chạy `npm install` -> Chạy `npm test` ra kết quả Baseline: 47 pass. Tiến hành code.

# Constraints

- 🚫 KHÔNG tiếp tục tạo worktree local nếu file đó đang không được đưa vào `gitignore` (Trừ khi thư mục ở Global scope).
- 🚫 KHÔNG giả định worktree path cứng ngắc mà bỏ qua tiến trình priority.
- Luôn kiểm tra Test Baseline.
