# Audit Severity Guide — Content Forge

## Phân loại mức độ nghiêm trọng

| Level | Label | Định nghĩa | Ví dụ | Action |
|---|---|---|---|---|
| 🔴 | **Critical** | Sai thật, mâu thuẫn logic, vi phạm compliance, data sai | Số liệu sai, acronym không khớp, thiếu disclaimer bắt buộc | FIX NGAY, block ship |
| 🟡 | **High** | Thiếu sót quan trọng, pacing issue, structural problem | Thiếu backup plan, section quá dài, missing citation | FIX trước ship |
| 🟡 | **Medium** | Inconsistency nhỏ, could-be-better | SFX không đồng nhất, tone shifts, formatting | FIX nếu dễ |
| 🟢 | **Low** | Nice-to-have, cosmetic | Thêm analogy, rephrase cho mượt hơn | Optional |

## Quy tắc fix

1. **Fix theo severity** — 🔴 trước, 🟡 tiếp, 🟢 nếu dễ
2. **Minimal impact** — chỉ sửa đúng chỗ, không touch nội dung khác
3. **Trace to finding** — mỗi edit trace về audit item cụ thể
4. **Version bump** — mỗi round fix = +0.1 version (v1.0 → v1.1)
5. **Verify fix** — round sau phải confirm "✅ Fixed" cho mỗi item

## Khi nào escalate cho user

- Round 3 vẫn < 9.0
- 🔴 Critical cần subject-matter input
- Hai agent cho score mâu thuẫn (1 cho 5, 1 cho 9 cùng mục)
- User data input không đủ để fix
