# Goal

> **Use this skill when:** git push, PR creation, branch management, or commit cleanup

Tối ưu hóa quy trình sử dụng git để đảm bảo các commit sạch sẽ (clean), quản lý branch chuẩn mực và push code lên remote (GitHub/GitLab) an toàn theo kiến trúc của ABM Workforce.

# Instructions

1. **Pre-Push Checklist:** Trước khi commit/push, xác minh 6 bước: VERIFY (tests pass?), CLEAN (git status?), COMMIT (convention?), BRANCH (đúng dòng?), REVIEW (tự check diff?), PUSH (đúng remote?).
2. **Commit Convention:** Sử dụng chuẩn Angular/Conventional Commits: `<type>(<scope>): <description>`. VD: `feat(auth): add login form`, `fix(api): handle timeout error`.
3. **Branch Strategy:** Không push code nháp hoặc code dev trực tiếp vào `main`. Mọi code dev nằm ở `develop` hoặc feature branch `fix/*`, `feat/*`, `skill/*`.
4. **Thao tác Push:** 
   - Feature Push: Tạo branch -> Dev -> Verify -> Push + PR.
   - Batch Push: Dùng `git rebase -i` (squash) nếu feature yêu cầu quá 3 commits nhỏ lẻ.
5. **Verified Evidence:** Bắt buộc dùng skill `evidence-driven-verification` TRƯỚC khi gõ lệnh push.

# Examples

## Ví dụ 1: Commit thêm skill
- `git add -A`
- `git commit -m "skill(abm-review): add 8-persona weighted scoring"`
- `npm test` >> Pass.
- `git push -u origin skill/abm-review` -> Mở PR.

# Constraints

- 🚫 Không BAO GIỜ push lên `main` nếu branch đó được configure bảo vệ bằng CI (PR only).
- Lọc kỹ `.gitignore`, cấm push các file `.env`, keys, secrets, hoặc các folder như `_abm-output/`, `.worktrees/`.
- Cấm force push mà không có confirm 2 lần.
