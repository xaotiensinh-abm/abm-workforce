# Examples

## Example 1: Customer support ops twin
Input gồm SOP xử lý ticket, policy refund, báo cáo tuần, meeting notes.
Output nên mạnh ở process catalog, escalation rules, metric cadence, role handbook.

## Example 2: Sales ops twin
Input gồm playbook CRM, report pipeline, memo forecast, checklist hygiene.
Output nên mạnh ở decision rules, process system, review cadence và failure points.

## Example 3: Founder office twin
Input gồm memo điều hành, planning deck, weekly review note, report template.
Output nên mạnh ở operating principles, decision layers, artifact cadence và open questions.

## Example 4: Real estate ops twin
Input gồm quy trình lead, script tư vấn, policy booking, checklist hồ sơ, báo cáo booking và meeting notes.
Output nên mạnh ở lead-to-booking-to-contract, vai trò sale/admin/pháp lý, exception path và metric cadence.

## Example 5: Dental clinic ops twin
Input gồm quy trình đặt lịch, xác nhận lịch, tư vấn điều trị, hướng dẫn sau điều trị, báo cáo no-show và tái khám.
Output nên mạnh ở patient journey, role handbook, capacity rule, recall cadence và service recovery.

## Example 6: SME operating twin
Input gồm memo founder, checklist vận hành, báo cáo tuần, quy định duyệt chi, onboarding note.
Output nên mạnh ở operating model, decision rules, meeting brief, onboarding và remediation plan.

## Example 7: Primary project sales
Input gồm giỏ hàng, booking policy, sales launch brief, daily booking report.
Output nên mạnh ở inventory control, booking rule, launch cadence, owner và escalation khi giữ chỗ ngoại lệ.

## Example 8: Brokerage resale
Input gồm quy trình sourcing hàng, script xem nhà, mẫu đặt cọc, checklist đàm phán.
Output nên mạnh ở sourcing, matching, viewing, negotiation, deposit handling và client follow-up.

## Example 9: Multi-chair dental clinic
Input gồm rule phân ghế, lịch bác sĩ, báo cáo no-show, checklist vô trùng, họp capacity.
Output nên mạnh ở chair utilization, turn-time, no-show recovery, sterilization handoff và staff coordination.

## Example 10: Trading-distribution SME
Input gồm quy trình mua hàng, tồn kho, giao hàng, công nợ, đổi trả và báo cáo tuần.
Output nên mạnh ở procurement-to-delivery flow, stock control, aging debt, exception path và reporting cadence.

## Example 11: Real estate orchestration
Input gồm launch SOP, booking policy, hồ sơ pháp lý giao dịch, timeline thanh toán và weekly legal-sales sync.
Output nên mạnh ở orchestration map giữa `primary-project-sales-vi-v1` và `legal-transaction-ops-vi-v1`, làm rõ booking-to-legal handoff, exception path, approval và metric chung.

## Example 12: Dental orchestration
Input gồm capacity board, lịch bác sĩ, recall plan, implant consent, báo cáo tiến độ treatment plan.
Output nên mạnh ở orchestration giữa `multi-chair-clinic-vi-v1` và `implant-ortho-heavy-vi-v1`, làm rõ clash giữa chair capacity và treatment cadence dài hạn.

## Example 13: SME orchestration
Input gồm SOP order-to-cash, debt review, memo founder, weekly review note và approval matrix.
Output nên mạnh ở orchestration giữa `trading-distribution-vi-v1` và `sme-ops-vi-v1`, làm rõ governance layer, owner, approval và review cadence.
