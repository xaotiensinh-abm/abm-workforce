# Goal

> **Use this skill when:** parallel task execution with independent sub-agents

Thực thi kế hoạch qua việc điều phối các subagent độc lập, với quy trình review 2 bước (spec compliance -> code quality) để đảm bảo chất lượng và tốc độ lặp lại mà không loãng ngữ cảnh.

# Instructions

1. **Phân rã (Per Task):** Trích xuất đầy đủ văn bản và ngữ cảnh của task từ plan.
2. **Implementer Subagent:** Gửi prompt (sử dụng `./implementer-prompt.md`). Nếu agent hỏi, trả lời rõ ràng. Nếu agent gặp status `BLOCKED`, cung cấp thêm ngữ cảnh hoặc chia nhỏ task. Tôn trọng report `DONE_WITH_CONCERNS`.
3. **Spec Reviewer Subagent:** Gửi prompt (sử dụng `./spec-reviewer-prompt.md`). Nếu báo chưa đủ spec => quay lại Implementer để sửa và review lại. Không bỏ qua.
4. **Code Quality Reviewer Subagent:** Gửi prompt (sử dụng `./code-quality-reviewer-prompt.md`). Nếu báo lỗi thiết kế/magic numbers => Implementer sửa và review lại.
5. **Đánh dấu hoàn thành:** Qua 2 vòng review, mark task done trong TodoWrite.
6. **Final:** Nếu duyệt xong hết task, dispatch final code reviewer trước khi kết thúc development branch bằng `finishing-a-development-branch`.
7. **Lựa chọn Model (nếu có thể setup):**
   - Task cơ học (1-2 file): Model nhanh, rẻ.
   - Thao tác tích hợp, debug: Model tiêu chuẩn.
   - Design, Architecture: Model thông minh nhất.

# Examples

- **Khởi động Subagent:** Cung cấp plan. Gửi Task 1 qua `implementer-prompt.md`. Agent làm xong tự review và commit.
- **Spec Reviewer:** Fail do thiếu progress bar báo cáo, báo Implementer sửa. Sau sửa -> Spec Reviewer pass.
- **Code Quality Review:** Fail do magic number (100). Implementer xuất `PROGRESS_INTERVAL`. -> Pass. Tính là kết thúc task 1.

# Constraints

- 🚫 KHÔNG BAO GIỜ bỏ qua review (kể cả spec compliance và code quality). Spec phải trước Code.
- 🚫 Không chạy nhiều implementer subagent song song (gây conflict nhánh hiện tại).
- Không tự đẩy Subagent đi tự đọc file plan chung chung, Orchestrator phải trích xuất và cung cấp toàn bộ đoạn text.
