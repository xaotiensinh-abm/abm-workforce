# Goal

> **Use this skill when:** verifying work before claiming completion

Kiếm chứng mọi tác vụ bằng bằng chứng kiểm soát cứng tự động hóa ngay trước khoảnh khắc được xưng "hoàn thành" hay có bất kỳ khẳng định thành công nào.

# Instructions

Trình tự hoạt động the "Gate Function":
1. **IDENTIFY:** Xác định lệnh CLI nào đủ quyền hạn chuyên môn chứng minh tính chính xác? (VD: `npm test`, `cargo build`, `eslint .`).
2. **RUN:** Dành thời gian chạy fresh (mới gọi lại toàn bộ) lệnh đó ở phiên bản code hiện tại.
3. **READ:** Đọc toàn bộ đầu ra (Output). Scan tới mã Exit Code và count pass/fail.
4. **VERIFY:** Kết quả xác thực liệu có 100% qua ải và liên kết mật thiết tới claim không.
5. **CLAIM LAST:** CHỈ khi có bằng chứng đó, Agent mới được kết luận là Finished/Passed kèm log cụ thể để report cho người dùng.

# Examples

- **Báo cáo chuẩn chỉ:**
  - ✅ [Run lệnh TDD Test] [Đọc log Test: "34/34 passing"] -> Agent Report: "All tests pass successfully".
  - ✅ Sửa xong Build lỗi. Run lại Build Exit Code 0. Report "Build passes."
- **Nghiêm cấm lấp liếm:**
  - ❌ (Nhìn source code) "Đã pass, file ok." -> Không có bằng chứng.
  - ❌ (Mới Linter xong, quên build): "Đã xong, Build passing."
  - ❌ Test chạy 1 lần qua, rồi bỏ đó.

# Constraints

- 🚫 Ngữ pháp cấm kị: Không dùng đại từ suy xét nhạy cảm nhắm mắt như (should work, probably, seems fixed, must be exactly what we need).
- Không được tin Agent/Subagent nói mà không đọc diff và build/test file thay đổi.
- Bằng chứng là luật, Code là luật. Partial check không chứng tỏ tổng thể.
