# Goal

> **Use this skill when:** executing an approved implementation plan step by step

Thực thi kế hoạch (implementation plan) đã viết sẵn một cách hệ thống, với các điểm kiểm tra định kỳ (review checkpoints), đảm bảo không bỏ sót bước nào.

# Instructions

1. **Khởi tạo:** Thiết lập workspace riêng biệt (vd: git worktree) nếu cần thiết. Đọc kỹ toàn bộ nội dung file kế hoạch.
2. **Review phê phán:** Đánh giá độ khả thi. Nếu có lỗ hổng hoặc điều chưa rõ ràng, DỪNG LẠI và hỏi (Ask for clarification) người dùng. KHÔNG tự đoán.
3. **Thực thi tuần tự:** Đánh dấu từng task là `in_progress`. Làm đúng theo mô tả trong kế hoạch, không thêm bớt (đặc biệt là YAGNI).
4. **Xác minh nội bộ:** Sau mỗi task, bắt buộc phải chạy lệnh kiểm tra (tests/build/lint) trước khi đánh dấu `completed`.
5. **Điểm kiểm toán (Checkpoints):** Tùy độ dài kế hoạch, dừng lại sau mỗi batch (VD: 3 tasks) để báo cáo và nhận review (sử dụng subagent hoặc hỏi người dùng).
6. **Hoàn thành:** Sau khi xong toàn bộ, sử dụng skill `finishing-a-development-branch` để gộp code hoặc tạo Pull Request.

# Examples

## Ví dụ 1: Thực thi Plan tạo Auth Module
- Review Plan: Phát hiện thiếu thông tin về token expiration -> Hỏi user.
- Tự động tick các task: `[x] Tạo file service`, `[x] Viết unit tests`.
- Checkpoint: Chạy `npm test` thành công -> Báo cáo xong batch 1.

# Constraints

- 🚫 Dừng ngay lập tức nếu test fail liên tục, thiếu dependency, hoặc instruction sai. KHÔNG cố gắng ép qua lỗi.
- 🚫 KHÔNG BAO GIỜ thực thi thẳng trên nhánh `main`/`master` mà chưa có sự đồng ý rõ ràng.
- Ghi lại mọi tiến độ trực tiếp vào file plan (Tick box) hoặc báo cáo qua terminal.
