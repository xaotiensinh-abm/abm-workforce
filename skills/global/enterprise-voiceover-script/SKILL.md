# Goal

> **Use this skill when:** e-learning voiceover scripts or broadcast narration

Sinh kịch bản voice-over enterprise-grade cho bài giảng điện tử, đạt chất lượng
broadcast từ lần xuất bản đầu tiên thông qua quy trình **Write → Audit → Fix → Loop**.
Mục tiêu: score >90/100 từ hệ thống đánh giá 12-agent adversarial audit.

# Instructions

## 📐 Phase 0: Thu thập thông tin (INTAKE)

Trước khi viết BẤT KỲ dòng script nào, PHẢI có đủ 8 thông tin:

| # | Thông tin | Bắt buộc | Ví dụ |
|---|---|---|---|
| 1 | **Tên bài** + số thứ tự | ✅ | "Bài 3: Prompt Engineering" |
| 2 | **Thời lượng tổng** | ✅ | 90 phút |
| 3 | **Mục tiêu đào tạo** | ✅ | 1-3 learning objectives (Bloom levels) |
| 4 | **Nội dung trọng tâm** | ✅ | Danh sách 3-5 chủ đề chính |
| 5 | **Đối tượng** | ✅ | "1.600 CBCNV BSR" |
| 6 | **Ngữ cảnh doanh nghiệp** | ✅ | Ngành, quy mô, thuật ngữ riêng |
| 7 | **Bài trước/sau** | 🟡 | Để tạo recap & preview liên kết |
| 8 | **Yêu cầu đặc biệt** | 🟡 | TGĐ endorsement, case study bắt buộc |

Nếu thiếu bất kỳ mục ✅ nào → HỎI TRƯỚC, không đoán.

## 📝 Phase 1: Thiết kế cấu trúc (OUTLINE)

### 1.1 Tính toán thời lượng

```
Tốc độ đọc chuẩn: 140 từ/phút (tiếng Việt, giọng trang trọng)
Voice time = 55-65% tổng thời lượng
Dead time = 15-25% (demo, thực hành, quiz, poll)
Buffer = 10-20% (facilitator, sự cố kỹ thuật)

Ví dụ: Bài 90 phút → Voice ~50' (~7.000 từ) + Dead ~22' + Buffer ~18'
```

### 1.2 Cấu trúc slide chuẩn

Mỗi bài PHẢI có 4 phần:

```
MỞ ĐẦU (5-8% thời lượng):
├── S0: Lời mở đầu Lãnh đạo (video/audio — 30s)
├── S1: Chào mừng + Recap bài trước + Mục tiêu bài này
└── S2: Hook (case study, thống kê gây sốc, demo visual)

THÂN BÀI (65-75% thời lượng):
├── Phần 1: Lý thuyết nền tảng + Ví dụ + Demo
├── Phần 2: Kỹ thuật/Quy trình + Demo + Thực hành nhóm
└── Phần 3: Nâng cao/Tích hợp + Bài tập cá nhân

KẾT THÚC (12-15% thời lượng):
├── S(n-4): Recap N trụ cột
├── S(n-3): Cheat Sheet / Takeaway (in được, dán bàn)
├── S(n-2): FAQ (3-5 câu thường gặp nhất)
├── S(n-1): Thông điệp kết thúc + Preview bài tiếp
└── S(n): Tài liệu tham khảo (slide tĩnh)

BẢNG TỔNG HỢP KỸ THUẬT (cuối file):
└── Table: Slide | Thời gian | Giây | Từ voice | Đọc @140w/m | Dead time | Margin
```

### 1.3 Quy tắc thiết kế Gagné 9 Events

| Event | Áp dụng trong script |
|---|---|
| 1. Gain attention | S2: Hook (case study shock, demo visual) |
| 2. Inform objectives | S1: "Sau X phút, anh chị sẽ..." |
| 3. Stimulate recall | S1: Recap bài trước |
| 4. Present content | Thân bài: lý thuyết + ví dụ |
| 5. Provide guidance | Demo live + analogies ngành |
| 6. Elicit performance | Thực hành nhóm + cá nhân |
| 7. Provide feedback | Debrief sau thực hành |
| 8. Assess performance | Quiz/Poll/Bài tập |
| 9. Enhance retention | Cheat Sheet + "khảo sát sau 2 tuần" |

## ✍️ Phase 2: Viết kịch bản (DRAFT)

### 2.1 Format mỗi slide

```markdown
### 🎬 S[N] — [Tên slide] | [Thời gian] | ~[X] từ | [Y]s voice

*[♪ Nhạc/SFX cue]*

[Nội dung voice-over — viết đúng như người đọc sẽ nói]

*[nhịp 2s]*  ← breath mark

[Tiếp tục nội dung]

*[beat 3s]*  ← longer pause between sections
```

### 2.2 Quy tắc viết voice-over enterprise

| Rule | Mô tả |
|---|---|
| **Giọng điệu** | Trang trọng nhưng gần gũi. "Anh chị" không "bạn". Không đùa cợt. |
| **Mật độ từ** | Tối đa 350 từ/slide @3 phút. Nếu >350 → chia slide |
| **Breath marks** | `*[nhịp 2s]*` sau mỗi 100-120 từ hoặc chuyển ý |
| **Beat marks** | `*[beat 3s]*` giữa các phần lớn |
| **SFX cues** | `*[🔔 Ding]*` khi bắt đầu framework/list. `*[♪ Transition]*` giữa phần |
| **Demo backup** | Mỗi demo PHẢI có "(Backup: nếu demo lỗi, hiện screenshots)" |
| **Localization** | Thuật ngữ EN giải thích inline. Ví dụ doanh nghiệp cụ thể |
| **Framework ghi nhớ** | Mỗi phần chính có 1 mnemonic (viết tắt, số, acronym) |
| **Cross-reference** | Liên kết với bài trước/sau: "Như anh chị đã học ở Bài N..." |
| **Closing connection** | Kết bài nối về giá trị/thành tích cốt lõi của doanh nghiệp |
| **TGĐ endorsement** | S0: 15-30s video/audio từ Lãnh đạo cao nhất — tạo authority |
| **Citations** | Mọi số liệu, case study PHẢI có nguồn (năm + tổ chức) |

### 2.3 Bảng tổng hợp kỹ thuật

Khi viết xong, PHẢI tạo bảng cuối file:

```
| Slide | Thời gian | Giây | Từ voice | Đọc @140w/m | Dead time | Margin | ✓ |
```

Kiểm tra:
- Tổng giây = thời lượng bài
- Tổng từ × 60/140 ≤ 65% tổng thời lượng
- Mọi slide có Margin dương (hoặc OK nếu thực hành)

## 🔬 Phase 3: Audit đa chiều (12-AGENT JARVIS AUDIT)

### 3.1 Bảng 12 Agent

Sau khi hoàn thành draft, chạy audit bằng 12 agent chuyên biệt:

| # | Agent | Vai trò | Trọng số |
|---|---|---|---|
| 1 | 🎯 Content Accuracy | Nội dung chuyên môn đúng, đủ, không mâu thuẫn | 10% |
| 2 | 🎓 Instruction Design | Gagné 9 Events, Bloom Taxonomy, learning arc | 10% |
| 3 | 🎙️ Audio/Production | SFX, breath marks, pacing, TGĐ slot, demo backup | 8% |
| 4 | 🏭 Domain Expert | Thuật ngữ ngành, ví dụ thực tế, case study chính xác | 10% |
| 5 | 🧪 QA Consistency | Số liệu, headings, numbering, cross-ref không mâu thuẫn | 10% |
| 6 | 🧠 Psychology | Hook hiệu quả, engagement curve, emotion arc | 8% |
| 7 | 🏢 L&D Director | Actionable takeaway, assessment rubric, follow-up | 8% |
| 8 | 🔄 Change Management | Tone "enabling" không "threatening", adoption framing | 8% |
| 9 | ⚖️ Legal/Compliance | Citations, disclaimers, regulatory references | 8% |
| 10 | 📺 Production Feasibility | Demo có backup? Slide count phù hợp? Timeline realistic? | 8% |
| 11 | ♿ Accessibility | Thuật ngữ EN giải thích? Jargon? Font-size friendly content? | 6% |
| 12 | 🇻🇳 VN Culture | Kính ngữ, cấu trúc Đảng/Đoàn, corporate hierarchy VN | 6% |

### 3.2 Scoring

Mỗi agent chấm 1-10. Weighted average = Final Score.

| Score | Grade | Action |
|---|---|---|
| ≥ 9.0 | ✅ PASS | Ship it. Nice-to-have optional |
| 8.0-8.9 | ⚠️ CONDITIONAL | Fix priority items → Re-audit |
| 7.0-7.9 | 🔴 REVISE | Major rewrite required |
| < 7.0 | ❌ REJECT | Restart from outline |

### 3.3 Output format audit

```markdown
# 🔬 JARVIS AUDIT — [Bài N] v[X] Round [R]
## Score: **[X.X]/10** | Target: >9.0

## 12-AGENT SCORECARD
| Agent | Score | Key Finding |
|---|---|---|

## PRIORITY ACTION ITEMS
| # | Severity | Issue | Slide | Fix |
|---|---|---|---|---|
| 🔴 Critical | ... | ... | ... |
| 🟡 High/Medium | ... | ... | ... |
| 🟢 Low | ... | ... | ... |

## STRENGTHS
1. ...
```

## 🔧 Phase 4: Chỉnh sửa (FIX)

### 4.1 Quy trình fix

1. Đọc Priority Action Items từ audit
2. Fix TẤT CẢ 🔴 Critical trước
3. Fix 🟡 High/Medium tiếp
4. 🟢 Low = nice-to-have, fix nếu dễ
5. Cập nhật bảng tổng hợp kỹ thuật nếu thay đổi timing
6. KHÔNG thay đổi nội dung không liên quan → minimal impact

### 4.2 Quy tắc fix

- Mỗi fix = 1 thay đổi nhỏ, verify trước khi fix tiếp
- KHÔNG batch fix mù → dễ introduce lỗi mới
- Version: v1.0 → fix → v1.1 → fix → v1.2...
- Document mỗi fix trong audit report "RESOLUTION STATUS"

---

## 🔄 Phase 5: Loop (RE-AUDIT → PASS/FIX)

```
┌────────────────────────────────────────────────┐
│         WRITE → AUDIT → FIX → RE-AUDIT         │
│         ↑                           │           │
│         └─── < 9.0 ─── SCORE ─── ≥ 9.0 → PASS │
│              (loop)                  (ship)      │
└────────────────────────────────────────────────┘
```

Re-audit format:

```markdown
# 🔬 JARVIS AUDIT — [Bài N] Round 2 (Post-Fix Verification)
## Score: **[X.X]/10 ✅** — TARGET >90 MET

## ROUND 1 ISSUES — RESOLUTION STATUS
| # | Issue | Status | Verification |
|---|---|---|---|

## ROUND 2 SCORECARD
| Agent | R1 → R2 | Assessment |

## REMAINING NICE-TO-HAVE
| # | Item | Impact | Effort |

**VERDICT: [X.X]/10 — PASS/FAIL**
```

Vòng lặp tối đa 3 rounds. Nếu Round 3 vẫn < 9.0 → escalate cho user review.

# Constraints

## Hard constraints — không thỏa hiệp

- 🚫 KHÔNG viết script mà không có intake (Phase 0)
- 🚫 KHÔNG skip audit — mọi script PHẢI qua ít nhất 1 round audit
- 🚫 KHÔNG ship script < 9.0/10 — loop bắt buộc
- 🚫 KHÔNG bịa case study/số liệu — PHẢI có citation (năm + nguồn)
- 🚫 KHÔNG dùng "bạn" trong script enterprise — luôn "anh chị"

## Quality guardrails

- Tốc độ đọc: 130-150 từ/phút (mặc định 140, điều chỉnh theo yêu cầu)
- Mật độ từ: ≤350 từ/slide @3 phút
- Breath marks: mỗi 100-120 từ
- Demo backup: 100% demo slots phải có fallback plan
- SFX consistency: đồng nhất across toàn bài
- Buffer: 10-20% thời lượng (quá 20% → cảnh báo)

## Scope

- Skill này CHỈ viết kịch bản voice-over text (markdown)
- KHÔNG tạo video, audio, slide design, hoặc LMS upload
- KHÔNG viết content cho interactive quiz (trừ question text)
- Output luôn là .md file — chuyển đổi format khác do user hoặc skill khác xử lý
