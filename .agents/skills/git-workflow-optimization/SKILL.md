# Git Workflow Optimization

## Overview

Tối ưu quy trình git push lên GitHub cho ABM-Vibecoding workspace.

**Nguyên tắc cốt lõi:** Push có hệ thống — clean commits, proper branching, verified code.

## Pre-Push Checklist

```
TRƯỚC KHI push BẤT KỲ code nào:

1. VERIFY  — Tests pass? Build ok? (fresh run)
2. CLEAN   — Git status clean? Staged đúng files?
3. COMMIT  — Message theo convention? Squash nếu cần?
4. BRANCH  — Đúng branch? Base branch updated?
5. REVIEW  — Self-review diff trước push?
6. PUSH    — Push lên đúng remote + branch
```

## Commit Convention

```
<type>(<scope>): <description>

Types: feat | fix | docs | style | refactor | test | chore | skill
Scope: skill-name | rules | config | docs

Ví dụ:
- feat(evidence-verification): add rationalization table
- skill(abm-review): add 8-persona weighted scoring
- docs: update Getting Started guide
- chore: cleanup unused files
```

## Branch Strategy

| Branch | Purpose | Push Policy |
|---|---|---|
| `main` | Production-ready skills | Protected, PR only |
| `develop` | Integration | Direct push OK |
| `skill/<name>` | New skill development | Direct push OK |
| `fix/<issue>` | Bug fixes | Direct push OK |

## Push Workflow

### Quick Push (đã verified)
```powershell
git add -A
git commit -m "feat(skill-name): description"
git push origin develop
```

### Feature Push (cần review)
```powershell
# 1. Tạo branch
git checkout -b skill/new-skill-name

# 2. Develop + commit
git add -A
git commit -m "skill(new-skill): initial implementation"

# 3. Verify
npm test  # hoặc verification command phù hợp

# 4. Push + PR
git push -u origin skill/new-skill-name
# Tạo PR qua GitHub UI hoặc gh cli
```

### Batch Push (nhiều changes)
```powershell
# Squash commits nếu > 3 commits cho cùng feature
git rebase -i HEAD~N
git push origin develop
```

## Red Flags

- ❌ Push mà chưa chạy tests
- ❌ Push trực tiếp lên `main` (dùng PR)
- ❌ Commit message không có type prefix
- ❌ Push files nhạy cảm (secrets, .env, API keys)
- ❌ Force push mà không confirm với team
