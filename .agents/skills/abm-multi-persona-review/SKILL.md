# ABM Multi-Persona Review

Đánh giá phản biện đa chiều sử dụng 8 personas chuyên biệt của ABM Workforce.

**Nguyên tắc cốt lõi:** Mỗi persona là một góc nhìn độc lập. Không persona nào có quyền phủ quyết — CEO (user) quyết định cuối cùng.

## Khi nào sử dụng

- Đánh giá architecture / design decisions
- Review code sau khi implementation hoàn tất
- Audit system integrity
- Phản biện business plan / strategy
- Trước khi deploy production

## Quy trình

### Bước 1: THU THẬP bằng chứng

```
Không dùng "chắc là" — đếm file, đo dòng code, đọc content thực tế.
```

### Bước 2: CHẤM ĐIỂM từng persona

Mỗi persona đánh giá 1-10 với evidence cụ thể:

```markdown
| Persona | Điểm | Phát hiện | Mức độ | Evidence |
|---|---|---|---|---|
| Hacker | 7 | SQL injection possible | 🔴 | [file:line] |
| Auditor | 6 | No audit log | 🟡 | Missing logging |
```

### Bước 3: PHÂN LOẠI phát hiện

- 🔴 **Nghiêm trọng** — fix ngay, chặn deploy
- 🟡 **Cần sửa** — fix trước release
- 🔵 **Cải thiện** — backlog
- ✅ **Tốt** — giữ nguyên

### Bước 4: ĐỐI CHIẾU chéo

Ma trận cross-reference: phát hiện của persona A ảnh hưởng persona B không?

### Bước 5: KẾ HOẠCH hành động

Ưu tiên: P0 (🔴 nghiêm trọng) → P1 (🟡 cần sửa) → P2 (🔵 cải thiện)

## Output Format

```markdown
## Tóm tắt (≤ 200 từ)
[CEO đọc đây trước]

## Bảng điểm
| Persona | Điểm | Top Finding |
|---|---|---|

## Điểm tổng: X.X/10 (weighted average)

## Hành động
| # | Priority | Action | Ai | Khi nào |
```

## Tích hợp Superpowers

**REQUIRED SUB-SKILL:** superpowers:verification-before-completion — verify evidence trước khi chấm điểm

**After review:** superpowers:writing-plans nếu cần plan fix issues

## Ví dụ: Review API Authentication Module

**Input:** Review module `src/auth/` (3 files, 450 lines)

**Thu thập:** `Get-ChildItem src/auth/ | Measure-Object` → 3 files, 450 lines

**Bảng điểm:**
| Persona | Điểm | Top Finding | Mức |
|---|---|---|---|
| 🔴 Hacker | 7 | JWT secret hardcoded trong config.js | 🔴 |
| 📊 Auditor | 6 | Không log failed login attempts | 🟡 |
| 💼 CEO | 9 | Auth module phục vụ 100% users | ✅ |
| 🏗️ Architect | 8 | Tách concerns tốt, thiếu rate limiting | 🟡 |
| 🔧 Pragmatist | 8 | Code clean, dễ maintain | ✅ |
| ⚔️ Competitor | 9 | Tương đương industry standard | ✅ |
| 🔄 Operator | 7 | Thiếu health check endpoint | 🟡 |
| 📚 New Hire | 8 | README rõ, thiếu API docs | 🔵 |

**Điểm tổng:** 7.8/10 (weighted)

**Hành động:**
| # | Priority | Action | Khi nào |
|---|---|---|---|
| 1 | P0 | Move JWT secret → env var | Ngay |
| 2 | P1 | Add login attempt logging | Sprint này |
| 3 | P1 | Implement rate limiting | Sprint này |

## Red Flags

- ❌ Chấm điểm mà không có evidence cụ thể
- ❌ Bỏ qua persona vì "không liên quan"
- ❌ Tất cả personas cho 10/10 (unrealistic)
- ❌ Output không có hành động tiếp theo
